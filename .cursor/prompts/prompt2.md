Создай React-компонент — 3D калькулятор кастомизации мерча.

Стек: @react-three/fiber, @react-three/drei, three, стандартный React (useState, Suspense).

## Макет страницы:
- Слева — вертикальное меню выбора **типа изделия**: зонт, кепка, футболка, кружка, пакет.
- Справа сверху — меню выбора **животного для принта** (контурный рисунок): котик, собачка,  крокодил, тигр. Рисунок должен быть **полым/контурным** (только линии, без заливки).(лежат в папке src/img)
- Справа снизу — выбор **цвета изделия** (5–6 кнопок: красный, синий, зелёный, чёрный, белый, жёлтый).
- Справа снизу — выбор **цвета фона сцены** через RGB-пикер (input type="color").

## Центр страницы — 3D сцена:
- Крупное **3D-изделие**, загружаемое из интернета, в зависимости от выбранного типа.
- Для загрузки используй `useGLTF` из `@react-three/drei` с готовыми моделями (найди публичные GLB-модели или используй плейсхолдеры от Ready Player Me / Sketchfab sample models).
- **ВАЖНО**: не создавай модели из примитивов (сфер/цилиндров), а загружай реальные 3D-модели по URL. Для каждого типа изделия должен быть свой URL модели. Если модель не загружается — показывай fallback.
- Оберни загрузку модели в `<Suspense>` с индикатором загрузки.
- К модели применяется:
  - **цвет изделия** — меняй материал модели (через traverse и обновление MeshStandardMaterial)
  - **принт** — накладывается поверх модели как отдельный плоский Mesh с текстурой-контуром, позиционируется в зависимости от типа изделия
- Модель можно **вращать мышкой** (OrbitControls с центром на модели).

## Принты (контурные изображения):
- **Вариант 1 (основной)**: загрузи готовые контурные изображения животных из интернета (найди PNG/SVG с прозрачным фоном и только контуром). Используй `useLoader` с `TextureLoader` для загрузки. Такжеготовые лежат  в папке src/img
- **Вариант 2 (резервный)**: если картинки не загрузились — нарисуй CanvasTexture с прозрачным фоном и контуром (stroke, без fill).
- **Дополнительно**: добавь кнопку "Загрузить свой принт" — пользователь может загрузить локальное изображение (PNG/SVG), и оно наложится на изделие поверх выбранного животного. Загруженное изображение должно сохранять прозрачность фона.
- Принт должен быть видимым на фоне цвета изделия.

## Фон сцены:
- Меняется через RGB-пикер на правой панели (scene.background).

## Взаимодействие:
- Любое изменение (тип изделия, животное, цвет изделия, фон) мгновенно обновляет сцену.
- Тип изделия загружает новую 3D-модель.
- Загруженный пользователем принт заменяет выбранное животное.

## Дополнительно:
- Добавь освещение: ambientLight + directionalLight + pointLight для хорошей видимости модели.
- Код должен быть полностью рабочим, самодостаточным, с импортами.
- Используй CSS Grid или Flex для позиционирования панелей поверх canvas.

## Архитектура (обязательное требование):
Раздели приложение на следующие компоненты в отдельных файлах:
- **LeftMenu.jsx** — левая панель с выбором типа изделия
- **RightMenu.jsx** — правая панель (выбор животного, цвет изделия, RGB-фон, кнопка загрузки своего принта)
- **ModelViewer.jsx** — центральная 3D сцена с моделью, принтом и вращением
- **App.jsx** — главный компонент, который объединяет всё и управляет состоянием (useState)

Все компоненты должны получать данные и колбэки через props.

## Итог:
Полностью рабочий проект с компонентной архитектурой. Код можно вставить в React-проект, и он заработает.
